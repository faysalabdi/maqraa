# arabic-xp

An Arabic **reading** app. Read real books in-app, tap any word for an instant
translation, save it to a spaced-repetition deck, and finish books — starting on a
short curated graded shelf and then bringing your own EPUBs.

> The motivation: reading Arabic is hard because one unknown word breaks the flow
> and you never finish the book. arabic-xp removes the dictionary friction (tap a
> word, get the lemma + meaning in context, keep reading) and measures real
> outcomes — books finished, words mastered, days in a row — instead of an
> abstract level.

---

## What it does

- **Read in-app, page by page.** Every book is split into chapters; each chapter
  is paginated into comfortable screen pages with a progress bar, text-size
  control, and paper/sepia/mint page tints.
- **Tap-to-translate.** Tap any Arabic word → its dictionary form (lemma), a
  contextual English gloss, part of speech, and an example sentence. Lookups are
  generated by Claude and cached globally, so a given word is only ever looked up
  once across all users.
- **Save → spaced repetition.** Saved words flow into an SM-2 review queue
  (`/review`) with a tap-to-flip flashcard and Again / Hard / Good / Easy grading.
  `/words` shows every saved word grouped by strength (New → Mastered).
- **Bring your own books.** Drop an `.epub` and it’s parsed in the browser into
  real chapters; one click of AI fills the difficulty band, genre, and blurb, and
  regroups badly-split EPUBs into proper chapters.
- **Optional comprehension test.** Finishing a book completes it; a Claude-graded
  whole-book test is available but never required.

## Progression model

There is **no level ladder**. Progress is measured by **outcomes**:

- **Books finished** — reaching the last page of the last chapter completes a book.
- **Words mastered** — cards that graduate in the SRS deck.
- **Streak** — consecutive days with any reading/review activity.

`books.level` still exists but is only an **advisory difficulty band** used to sort
shelves (1 ≈ absolute beginner … 9 ≈ scholarly). It does not gate anything.

**Content model:**

- A small **curated on-ramp** ships in the seed (an original graded reader plus
  public-domain hadith collections and Kalila wa Dimna). Everyone starts here.
- After finishing **2 books**, a reader unlocks **uploading their own** books,
  which are **private to them** (`books.owner_id`). Admins (via `ADMIN_EMAILS`)
  add **public** catalogue books (`owner_id = null`).

This is deliberate: curation solves the cold-start/difficulty problem for
beginners; BYO gives advanced readers freedom once they’ve built a base. AI
difficulty grading only *shelves* a book — it never gates progress, so a noisy or
gamed grade is harmless.

## Stack

- **Next.js 15** App Router · TypeScript strict · React 19
- **Tailwind v4** (design tokens are CSS variables in `app/globals.css` `@theme`) ·
  Framer Motion · lucide-react
- **Supabase** Auth + Postgres + RLS
- **Drizzle ORM** — typed schema in `lib/db/schema.ts`, migrations in `db/migrations/`
- **Anthropic SDK** (Claude) — word lookups, chapter quizzes, whole-book tests,
  and upload analysis, with ephemeral prompt caching on system blocks
- **jszip** — client-side EPUB parsing (no server upload pipeline)
- **Vercel** deployment

## Local development

```bash
pnpm install
cp .env.example .env.local      # Supabase URL + keys, DATABASE_URL/DIRECT_URL, ANTHROPIC_API_KEY
pnpm db:push                    # apply the Drizzle schema to your DB
pnpm db:seed                    # seed difficulty bands, the curated on-ramp books + chapters, achievements
pnpm dev                        # http://localhost:3000
```

**Connecting to Supabase from a laptop:** the direct DB host
(`db.<ref>.supabase.co`) is IPv6-only and often won’t resolve locally. Point
`DATABASE_URL` / `DIRECT_URL` at the **session pooler**
(`aws-<n>-<region>.pooler.supabase.com:5432`, user `postgres.<ref>`) instead.

**Becoming an admin / uploading curated books:** add your email to
`ADMIN_EMAILS` (comma-separated) in `.env.local`. Admin uploads become public
catalogue books; everyone else’s uploads are private after they’ve finished 2 books.

Scripts: `pnpm db:generate` (diff schema → SQL), `pnpm db:studio`, `pnpm test`
(Vitest), `pnpm lint`, `pnpm build`.

## Uploading a book (the EPUB pipeline)

`/upload` (“Your library”) → drop an `.epub`:

1. **Parse in the browser** (`lib/books/epub.ts`, jszip). Reads the OPF for
   title/author, then the book’s own table of contents (EPUB3 `nav` or EPUB2
   `toc.ncx`) to find **real chapter boundaries** and merges the spine pages
   between them — so per-page EPUBs don’t explode into hundreds of fragments.
   Falls back to one chapter per spine document when there’s no usable TOC.
2. **Auto-fill with AI** (optional, one button — `lib/ai/book-analyze.ts`). Claude
   reads only short excerpts (an opening sample + each page’s first lines) and
   returns a difficulty band, genre, blurb, and a contiguous page→chapter grouping
   that drops front/back matter (cover, copyright, ISBN, TOC, about-the-author).
   Whole books are never sent to the model.
3. **Preview & save.** Review/edit chapter titles, preview text, then save —
   `createBookWithChapters` writes the book + chapters and stamps `owner_id`.

Slugs are auto-slugified, so a title like “Animal Farm” becomes `animal-farm`.

## Reading & completion lifecycle

- Open a book → `/book/[slug]/read/[chapter]`. The reader paginates the chapter,
  marks it read on the last page, and on the **last chapter** completes the book
  (`markChapterRead` in `server/actions/chapters.ts`) with length-scaled XP.
- `user_books.status`: `in_progress` → `completed` (read-through). The optional
  test records a `comprehension_attempts` score; failing it never un-completes a
  finished book.
- Visibility: private uploads (`owner_id` set) are only readable/testable by their
  owner — the catalogue, book page, reader, and test routes all enforce this.

## Routes

| Route | Purpose |
| --- | --- |
| `/` | Signed-out landing |
| `/path` | **Read** — Continue hero, “Start here” curated shelf, “Your library” |
| `/book/[slug]` | Book detail: cover, chapters, start reading, optional test |
| `/book/[slug]/read/[chapter]` | The page-by-page reader + tap-to-translate |
| `/book/[slug]/test` | Optional whole-book comprehension test |
| `/words` | Saved-word collection grouped by SRS strength |
| `/review` | SM-2 flashcard session |
| `/upload` | Add a book (EPUB importer + AI) — gated by books-finished |
| `/settings`, `/stats`, `/leaderboard`, `/achievements`, `/practice` | Secondary |

## Key files

- `components/reader/ChapterReader.tsx` — the reader
- `components/upload/AddBook.tsx` — the upload wizard
- `app/(app)/path/page.tsx` — the Read catalogue
- `lib/admin.ts` — `requireUploader` / `uploadUnlocked` / `booksFinished` gates
- `lib/srs/sm2.ts` — pure SM-2 (unit-tested) · `lib/xp/{rewards,grant}.ts` — XP math

## Conventions

- Arabic text uses the `font-arabic` utility and `dir="rtl"` (Noto Naskh Arabic,
  Amiri fallback — `lib/fonts.ts`).
- All in-app text is **public-domain or original**, or a reader’s own private
  upload. There is no public user-generated catalogue.
- No `tailwind.config.ts` — tokens live in `app/globals.css`.
- Vitest covers pure logic (SM-2, XP curve, slugify, EPUB splitter).
